Monolith        →  One big application
                   Everything together
                   Single deployment

Microservices   →  Many small services
                   Each does one job
                   Deploy independently


Monolith
  → Simple, fast to build
  → Hard to scale
  → One failure = all down
  → Good for small projects

Microservices
  → Complex, slow to set up
  → Scale independently
  → Isolated failures
  → Good for large projects
