3 Key Points to Remember
If you forget everything else, remember these 3 things:

Factory hides the creation logic — client just asks for what they need by type
Loose coupling — client depends on interface, not concrete classes
Real-world use — notification systems, payment gateways, JDK's


-----------------------------------------------------

Why Factory is Useful (Benefits)

"Factory gives us:

Loose coupling — client doesn't depend on concrete classes
Single responsibility — object creation is in one place
Easy to extend — add new types by changing only the factory
Easy to test — can swap factory in tests"
