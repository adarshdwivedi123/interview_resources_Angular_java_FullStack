HTTP protocol
--------------------
	1. Stands for Hypertext transfer protocol
	2. Http acts a mediator between client and server
	3. Http is a stateless protocol, it will treat every request as a new request
		(Har request ake new request hoti hais)

Note: To develop REST API we should know below details about HTTP protocol 

	1. HTTP methods
	2. HTTP status codes
	3. HTTP request
	4. HTTP response


	1. HTTP methods
----------------------------
		a. GET: to get data from server to client
		b. POST: to send data from client to server
		c. PUT: to update data at server
		d. DELETE: to delete data from sever

		E)PATCH: to update the data partiallys



Note: every REST API method/endpoint should bind to HTTP protocol
=> getTicketData() ----> HTTP GET       ------> @GetMapping
=> bookTicket(..,..) ----> HTTP POST     ------> @PostMapping
=> updateProuct(..)----> HTTP PUT      -------> @PutMapping
=> deleteProduct(..) --> HTTP Delete   -------> @DeleteMapping


HTTP status codes
-------------------------

Server will send HTTP status code to client in the response
Http status code will indicate how server processed our request

1XX (100 to 199) --> informational status code
2xx (200 to 299) --> SUCCESS status code (OK)
3xx (300 to 399) --> Redirectional 
4xx (400 to 499) --> client error
5xx (500 to 599) --> server error
