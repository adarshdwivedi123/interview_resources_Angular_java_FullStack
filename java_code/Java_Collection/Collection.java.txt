#)Comparator vs Comparable
1)Comparable
Used to define natural order for objects.

Class implements Comparable<T> and overrides compareTo() method.

Example: Strings naturally sorted alphabetically; Numbers naturally sorted numerically.


2)Comparator
When u want to change the natual ordering 
#)Use Comparator :when you want external/custom sorting without modifying the class.

-------------------------------------------------------------------------------------

#)Comparable is used by classes to say that they can be compared to something else, usually another instance of the same class. You use this when you want to be able to call myObj.compareTo(otherObj).

#)A Comparator is used to compare to objects in much the same way, but with an important distinction: a Comparator is a separate class from the classes being compared. You would use it like myComparator.compare(obj1, obj2).

So they do the same thing, just in slightly different ways. Comparable is usually more convenient if you're comparing a class you wrote yourself, meaning you can have that class implement Comparable. ComparaY tors are better when you don't want to or can't change the code of the class that you want to compare.
