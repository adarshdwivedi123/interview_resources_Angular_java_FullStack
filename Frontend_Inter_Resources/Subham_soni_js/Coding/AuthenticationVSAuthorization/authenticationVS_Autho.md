
Authentication and authorization are security features implemented by browsers

#)Authentication is the process of verifying a user's identity, 

while authorization determines what that user can access.
 Authentication always comes before authorization because you cannot 
 authorize a user before identifying them


 Airport security
 
When you go through airport security, you show your ID to authenticate your identity.
 Then, when you arrive at the gate, you present your boarding pass to the flight attendant to authorize you to board your flight.


==================================================================================
 Authentication ka matlab hota hai user ki identity verify karna. Jaise jab user login karta hai to system username aur password check karta hai ki user valid hai ya nahi.

Authorization ka matlab hota hai check karna ki authenticated user ko system me kya access milna chahiye. Ye usually user ke role ya permissions par depend karta hai

===================================IMP===========

Jab user login karta hai aur system username aur password verify karta hai, us process ko authentication bolte hain.
Login hone ke baad user kya kya kar sakta hai — jaise admin page access kar sakta hai ya nahi — usko authorization bolte hain.

 #)Refresh Token
 A refresh token is a special token that allows an application to get new access tokens without prompting the user.
