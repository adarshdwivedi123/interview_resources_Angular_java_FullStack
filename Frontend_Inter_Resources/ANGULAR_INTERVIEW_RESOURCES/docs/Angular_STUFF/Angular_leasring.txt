Observables vs Promises
Observables are asynchronous like promises, but the key distinction is that Observables can return multiple values over time, and promises simply return a single value.

Observables are not executed until we subscribe to them using the subscribe() method, and they can emit multiple events.
Promises handle a single event and are executed even if you do not attach a callback method.

#)Template-Driven Forms in Angular
A template-driven form is the simplest way to build a form in Angular. It uses Angular’s two-way data-binding directive
 (ngModel) to create and manage the underlying form instance.

#)Additionally, as the name suggests, a template form is mainly driven by the view component. So,
 it uses directives placed in HTML rather than TypeScript or JavaScript to manage the form.
 A template-driven form is asynchronous due to the use of “directives” because the creation of form controls is delegated to the declared directives (IoC).


When to Use a Template-Driven Form
It is essential to know when to use a template-driven form in your Angular app. You may use a template-driven form if:

Your form is small to medium scale (around 10 controls with no steps).
Your form utilizes the validation attributes integrated with HTML without complex validation.
Your form has basic requirements that can be managed within the template with minimal lines of code.



Reactive Forms in Angular
The second way to develop forms in Angular is to use reactive forms.
 Reactive forms utilize the component class to programmatically declare the form controls and the required validators synchronously.

 Reactive forms use classes such as FormGroup and FormControl to define groups and controls wired to the template.

If your form has a lot of controls with many groups (children) that require complex validation logic.

Interceptors are an excellent way to modify the HTTP request or response to make it more simple and understandable.
. They are implemented as services and are often used for tasks such as adding headers, handling authentication, or logging. 

How Does The Interceptor Work?
Assume we need to do some operation on each request sent through httpClient in the application.

For example, we must provide an auth token in each request so that the server can authenticate and deliver us the required answer. To do this, we will establish a service in between that will allow us to modify our httpRequest or httpResponse accordingly. 


How To Create A Simple HTTP Interceptor
Angular interceptor can be created using the following command:

ng generate interceptor ExampleInterceptor
#)An angular interceptor will look the same as a service with @Injectable(), which implements HttpInterceptor, which has a method called intercept that allows you to intercept the HTTP request and response.

import { Injectable } from '@angular/core';
import { HttpInterceptor, HttpEvent, HttpRequest, HttpHandler } from '@angular/common/http';
import { Observable } from 'rxjs';

@Injectable()
export class ExampleInterceptor implements HttpInterceptor {
  intercept(httpRequest: HttpRequest<any>, next: HttpHandler): Observable<HttpEvent<any>> {
      //any alteration in httpRequest can be done here
    return next.handle(httpRequest);
  }
}



Here, Angular gives a reference to the httpRequest object when the intercept() function is used. We can review and alter this request as required. When our logic is finished, we use next.handle and send the revised request back to the application.

function removeOppositeDirections(directions) {
    const oppositeMap = {
        "NORTH": "SOUTH",
        "SOUTH": "NORTH",
        "EAST": "WEST",
        "WEST": "EAST"
    };
    const stack = [];

    for (let direction of directions) {
        // If the current direction is the opposite of the last one in the stack, pop it
        if (stack.length > 0 && stack[stack.length - 1] === oppositeMap[direction]) {
            stack.pop();
        } else {
            stack.push(direction);
        }
    }

    return stack;
}

const inputDirections = ["NORTH", "EAST", "SOUTH", "WEST"];
const output = removeOppositeDirections(inputDirections);
console.log(output); // Output will be an empty array since opposite directions cancel each other out

async pipe  vs subscribe

In your Angular template, you can use the async pipe to directly display data from an Observable or Promise without manually subscribing to it. The async pipe subscribes to the observable and updates the view whenever new data is emitted.


The async pipe simplifies the management of subscriptions and avoids memory leaks, as it automatically unsubscribes when the component is destroyed.


Let’s say you have a component with an observable property, and you want to display its value in the template using the async pipe:

import { Component, OnInit } from '@angular/core';
import { Observable, interval } from 'rxjs';

@Component({
  selector: 'app-example',
  template: `
    <div>
      <p>Current Time: {{ currentTime$ | async | date:'medium' }}</p>
    </div>
  `,
})
export class ExampleComponent implements OnInit {
  currentTime$: Observable<Date>;

  ngOnInit() {
    // Create an observable that emits the current time every second
    this.currentTime$ = interval(1000).pipe(map(() => new Date()));
  }
}
In this example, we use the async pipe to display the current time from the currentTime$ observable. The async pipe automatically subscribes to currentTime$ and updates the view with the latest emitted value.

#)forkJoin
forkJoin is another powerful operator in RxJS used for combining multiple observables into a single observable, and emitting the last values of each observable when all observables complete.

This is particularly useful when you need to make multiple HTTP requests in parallel and wait for all of them to complete before taking further action.

We have an Angular component ExampleComponent which fetches data from two different APIs (https://api.example.com/data1 and https://api.example.com/data2) using HttpClient.
The getData() method creates two separate observables for each HTTP request.
We use forkJoin to combine these two observables into a single observable. It will emit an array containing the last value from each observable when both observables complete.
We subscribe to the combined observable and receive an array containing the data from both API calls. We can then process this data as needed.

import { Component, OnInit } from '@angular/core';
import { HttpClient } from '@angular/common/http';
import { Observable, forkJoin } from 'rxjs';

@Component({
  selector: 'app-example',
  templateUrl: './example.component.html',
  styleUrls: ['./example.component.css']
})
export class ExampleComponent implements OnInit {

  constructor(private http: HttpClient) { }

  ngOnInit(): void {
    this.getData().subscribe(([data1, data2]) => {
      console.log('Data from first API call:', data1);
      console.log('Data from second API call:', data2);
    });
  }

  getData(): Observable<any[]> {
    const request1 = this.http.get<any>('https://api.example.com/data1');
    const request2 = this.http.get<any>('https://api.example.com/data2');

    return forkJoin([request1, request2]);
  }
}

In Angular, preloading allows you to load lazy-loaded modules in the background while the user is interacting with the initial view of the application. This can significantly improve the perceived performance of your application by reducing the time it takes to navigate to subsequent views.

Angular provides several strategies for preloading modules:

No Preloading: This is the default behavior, where lazy-loaded modules are loaded only when requested by the user. No preloading strategy is employed, and modules are loaded on-demand.
PreloadAllModules: This strategy preloads all lazy-loaded modules in the background after the initial module is loaded. This ensures that all modules are available immediately when requested by the user.
PreloadSelectedModules: With this strategy, you can specify which lazy-loaded modules should be preloaded. You can define a custom preload strategy by implementing the PreloadingStrategy interface and specifying the modules to preload based on your criteria.
To configure preloading in your Angular application, you can use the PreloadAllModules or PreloadSelectedModules strategy in your RouterModule.forRoot() method.



Custom Preload Method
With this configuration, all lazy-loaded modules will be preloaded in the background after the initial module is loaded.

If you want to use a custom preloading strategy (PreloadSelectedModules), you can implement the PreloadingStrategy interface and define your own logic for determining which modules to preload.



import { Injectable } from '@angular/core';
import { PreloadingStrategy, Route } from '@angular/router';
import { Observable, of } from 'rxjs';

@Injectable({
  providedIn: 'root'
})
export class CustomPreloadingStrategy implements PreloadingStrategy {
  preload(route: Route, load: () => Observable<any>): Observable<any> {
    if (route.data && route.data.preload) {
      return load();
    } else {
      return of(null);
    }
  }
}

#)Setting Query Parameters
 To set query parameters, you can use the Router service to navigate to a route with parameters. 

1. Using RouterLink
If we are navigating through a template, we can use the [routerLink] directive along with [queryParams]:

ex:<a [routerLink]="['/your-route']" [queryParams]="{ param1: 'value1', param2: 'value2' }">Link</a>


2.Using Router.navigate
If you are navigating programmatically, we can use the navigate method of the Router service:
import { Router } from '@angular/router';

constructor(private router: Router) { }

navigateWithParams() {
  this.router.navigate(['/your-route'], { queryParams: { param1: 'value1', param2: 'value2' } });
}

Retrieving Query Parameters
To retrieve query parameters in a component, you can use the ActivatedRoute service.

3. Using ActivatedRoute Snapshot
This is useful when you only need to get the parameters once and don't expect them to change.

typescript
Copy code
import { ActivatedRoute } from '@angular/router';

constructor(private route: ActivatedRoute) {
  const params = this.route.snapshot.queryParams;
  console.log(params.param1); // Output: value1
}
4. Using ActivatedRoute Observable
This is useful when you expect query parameters to change while the component is still active.

typescript
Copy code
import { ActivatedRoute } from '@angular/router';

constructor(private route: ActivatedRoute) {
  this.route.queryParams.subscribe(params => {
    console.log(params.param1); // Output: value1 (and updates when params change)
  });
}
