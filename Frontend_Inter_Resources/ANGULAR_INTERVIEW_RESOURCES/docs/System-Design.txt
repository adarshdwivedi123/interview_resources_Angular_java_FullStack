Distributed System  and three Priccipal
these are three principla we should follow to make robust and scalable system

1) Single Responsibilty Principle
if a component performs multiple tasks break it down into 
smaller components ,performing a single task.
each component should ideally have a single responsibilty.

2) No Single point  of failure Principle
In our system find all the single point of failures and add 
redundant copies of them so that if one copy falls other can 
take over.

3) No bottle-neck Principle
whenever we design a system think of the data flowing through
it find out the compoonent which  can slow the overall performance 
of the system increase their performae either by vertical scaling
or horizontal scaling

#vertialScaling :vertical scaling is upgrading the existing hardware 
or software.
#horizontalScaling:Means adding multiple system to imporve the 
performance.
